# Задание 1. Анализ и планирование

### 1. Описание функциональности монолитного приложения

**Управление отоплением:**

- Пользователи могут удаленно управлять отоплением в доме
- Система поддерживает подключение специальных датчиков и реле

**Мониторинг температуры:**

- Пользователи могут удаленно проверять температуру в доме
- Система поддерживает подключение специальных датчиков температуры

### 2. Анализ архитектуры монолитного приложения

Архитектура приложения представляет собой монолит на Java с СУБД Postgres. Всё синхронно. Никаких асинхронных вызовов, микросервисов и реактивного взаимодействия в системе нет. Всё управление идёт от сервера к датчику. Данные о температуре также получаются через запрос от сервера к датчику.

### 3. Определение доменов и границы контекстов

1) Домен управления устройствами
    - контекст: управления отоплением
2) Домен мониторинга
    - контекст: мониторинг температуры

### **4. Проблемы монолитного решения**

- позволяет лишь управлять отоплением в доме и проверять температуру
- каждая установка требует выезда специалиста по подключению системы отопления в доме к текущей версии системы 
- самостоятельно подключить свой датчик к системе пользователь не может
- поддерживается только специальные датчики и реле, не поддерживается подключение к экосистеме устройств партнёров по стандартным протоколам
- масштабируемость ограничена, так как монолит сложно масштабировать по частям
